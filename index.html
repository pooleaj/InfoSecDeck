Why Cybersecurity Is the Career Move of the Decade — And Every Tool You Need to Make It

The numbers don't lie. Over 4 million cybersecurity positions sit unfilled globally right now — and that gap isn't closing, it's widening. As ransomware groups, nation-state actors, and opportunistic hackers grow more sophisticated, organizations are desperate for talent. The result? A field where median compensation exceeds $119,000 per year, job growth outpaces nearly every other tech discipline at 33%, and practitioners at the senior level routinely clear $200K+. The cybercrime economy is projected to cost the world $9.8 trillion in 2025 alone. The people defending against it have never been more in demand — or better compensated.

The Salary Reality No One Talks About

Browse job postings and you'll find vague ranges like “$80K–$130K.” What those listings don't tell you is that a SOC Analyst in San Francisco with an AWS Security Specialty certification is clearing $145,000. A Senior IAM Engineer with a CISSP in New York? Closer to $190,000. Location, certifications, and specialization create a multiplier effect most candidates don't understand until they're already in the room negotiating.

There Isn't One Cybersecurity Career. There Are Many.

Most people think “hacker” when they hear cybersecurity. But the field spans multiple distinct disciplines: Security Operations, Identity & Access Management, Cloud Security, Application Security, Governance Risk & Compliance, Offensive Security, Digital Forensics & Incident Response, Security Engineering, and the rapidly emerging field of AI Security. Each has its own tools, career trajectory, and compensation ceiling — and the right path for you depends on where your interests and existing skills intersect.

The Career Ladder maps every role across 6 tiers from Help Desk to CISO — with real salary bands, required skills, and adjacent paths. The Security Domains Explorer breaks down each specialty in depth so you can find your fit before you spend money on a certification that won't serve your actual goal.

Certifications: The Fastest Shortcut That Isn't Actually a Shortcut

Hiring managers use certifications as an HR filter before your resume reaches a human. A Security+ clears the first ATS screening. A CISSP signals you're ready for senior leadership. An OSCP tells every red team hiring manager that you can actually hack — not just study how to. But the order you pursue certs matters enormously. The wrong sequence wastes both time and money.

The Candidates Who Get Hired Have Proof of Work

Certifications open doors. But what closes offers is demonstrable, hands-on experience. In a field where everyone claims to know Splunk, the candidate who built a detection rule that caught a real pass-the-hash attack in their home lab wins. The candidate who can walk through their CTF methodology during a technical screen wins. Credentials tell them you studied. Labs and challenges show them you can execute.

Foundation: Virtualization Setup
Install your hypervisor before building any other lab
1
Check your hardware
You need at least 16GB RAM (8GB minimum, 2–3 VMs at once requires 16GB+). CPU: enable virtualization (Intel VT-x / AMD-V) in BIOS. Disk: 100GB+ free for VMs.
2
Install a hypervisor
Choose one: VMware Workstation Pro (now free for personal use — best performance) or VirtualBox (free, open source, cross-platform). Download from official sites only.
VMware: broadcom.com/support/vmware-by-broadcom
VirtualBox: virtualbox.org
3
Create your first VM: Kali Linux
Download the Kali pre-built VMware or VirtualBox image from kali.org. Import it directly — no manual install needed. Allocate 4GB RAM, 2 CPUs, 60GB disk.
kali.org/get-kali/ → Virtual Machine Images
4
Set up networking
Create a Host-Only network adapter for isolated lab traffic between VMs. Use NAT for internet access when updating tools. Never put vulnerable VMs on your main network.
5
Take snapshots
Before starting any lab exercise, take a snapshot of every VM. After breaking something (you will), just revert. This is the #1 habit that separates efficient lab users from frustrated ones.
Tools Used:VMware Workstation ProVirtualBoxKali Linux
🛡️
Defensive Lab: SIEM + Detection Engineering
Build a real detection environment with log collection, correlation, and alerting
1
Set up a Windows VM (log source)
Use a free Windows 10/11 evaluation ISO from Microsoft or use Windows Server 2022 (180-day eval). This will be your primary log-generating machine.
2
Deploy Sysmon for rich telemetry
Sysmon adds deep Windows event logging: process creation, network connections, file hashes, registry changes. Use the SwiftOnSecurity sysmon-config — the community standard configuration.
winget install Sysinternals.Sysmon
sysmon64.exe -accepteula -i sysmonconfig.xml
3
Install Splunk Free (SIEM)
Splunk Free gives 500MB/day of indexing — plenty for lab use. Install on your host or a dedicated Linux VM. Create a new index called "lab".
splunk.com/download → Free Trial → Install
splunk start
# Navigate to localhost:8000
4
Forward Windows logs to Splunk
Install the Splunk Universal Forwarder on your Windows VM. Configure it to forward Security, System, Application event logs plus Sysmon (Event ID 1, 3, 7, 11).
# inputs.conf on Windows forwarder:
[WinEventLog://Microsoft-Windows-Sysmon/Operational]
[WinEventLog://Security]
5
Write your first detection rule
Create a saved search in Splunk that alerts when PowerShell is launched by a non-admin process — a common attacker technique. This is your first real detection engineering exercise.
index=lab EventCode=1 Image="*powershell.exe"
ParentImage!="*explorer.exe"
| alert if count > 0
6
Alternative: Try Wazuh (free open-source)
Wazuh is a free, full-featured SIEM+XDR stack. Easier to deploy than Splunk for beginners. Includes pre-built MITRE ATT&CK correlation rules, FIM, and vulnerability scanning out of the box.
wazuh.com → OVA image → Import to VMware/VBox
Tools Used:Splunk FreeSysmonWazuhSecurity OnionUniversal Forwarder
🔴
Offensive Lab: Attack Practice
Set up a safe, isolated environment to practice penetration testing techniques
1
Install Kali Linux (attacker VM)
Download the pre-built Kali VM image from kali.org. Import to VMware or VirtualBox. Kali includes Nmap, Metasploit, Burp Suite, Wireshark, Hydra, and hundreds of other tools pre-installed. Update immediately.
sudo apt update && sudo apt full-upgrade -y
2
Set up a vulnerable target VM
Metasploitable 3 — Intentionally vulnerable Ubuntu/Windows VMs. Built specifically for Metasploit practice. Also try VulnHub machines for CTF-style challenges.
github.com/rapid7/metasploitable3
# or download pre-built from vulnhub.com
3
Configure isolated Host-Only networking
Put both VMs on the same Host-Only adapter (VMnet1 in VMware). This creates an isolated network — no internet access for target, preventing accidental exposure. Never put vulnerable VMs on NAT or bridged mode.
4
Run your first Nmap scan
Discover the target and enumerate open ports. Use this exact methodology for every engagement: ping sweep → port scan → service detection → OS fingerprint.
nmap -sn 192.168.56.0/24 # discover hosts
nmap -sV -sC -O 192.168.56.20 # full scan
nmap -p- --min-rate 5000 TARGET # all ports
5
Practice web application attacks with DVWA
Deploy DVWA (Damn Vulnerable Web App) via Docker on your Kali VM. Practice SQL injection, XSS, CSRF, command injection, and file upload vulnerabilities at adjustable difficulty levels.
docker run -d -p 80:80 vulnerables/web-dvwa
# Browse to http://localhost/login.php
6
Progress to TryHackMe and HackTheBox
After you're comfortable with local VMs, level up with TryHackMe (guided paths, beginner-friendly) and HackTheBox (realistic machines, OSCP prep). Both have free tiers.
Tools Used:Kali LinuxMetasploitNmapBurp SuiteDVWAVulnHub
🏢
Active Directory Lab
Build a realistic enterprise environment for IAM, red team, and detection practice
1
Download Windows Server 2022 evaluation
Microsoft offers free 180-day evaluation ISOs. No license key required. Create a new VM: 4GB RAM, 2 CPUs, 60GB disk. Install Server Core or Desktop Experience.
microsoft.com/en-us/evalcenter/evaluate-windows-server-2022
2
Promote to Domain Controller
Install the AD DS role and promote the server to a Domain Controller. Use domain name lab.local. This will install DNS automatically.
Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools
Install-ADDSForest -DomainName "lab.local"
3
Add a Windows 10/11 client VM
Create a second VM with Windows 10/11 evaluation. Join it to the lab.local domain. Create 3–5 standard domain users and 1–2 admin accounts. This mirrors a real enterprise environment.
Add-Computer -DomainName "lab.local" -Restart
4
Create deliberate misconfigurations for practice
These are the real learning exercises: set a user with adminCount=1, enable unconstrained delegation on a computer, add a user to Domain Admins incorrectly, use a weak Service Account password (for Kerberoasting practice).
5
Attack with BloodHound (from Kali VM)
Run BloodHound with SharpHound collector to map all AD relationships. Identify attack paths to Domain Admin. This teaches both offense (how attackers escalate) and defense (what to harden).
python3 bloodhound.py -u alice -p Password1 -d lab.local -c All
6
Detect the attacks in your SIEM
Enable advanced audit policies on the DC. Forward Security event logs to Splunk/Wazuh. Write detection rules for: Kerberoasting (Event 4769), Pass-the-Hash (Event 4624 LogonType 3), BloodHound enumeration (4662 repeated ACL access).
Tools Used:Windows Server 2022BloodHoundMimikatzImpacketPowerView
☁️
Cloud Lab: AWS + Azure Free Tier
Practice real cloud security skills without any upfront cost
1
Create an AWS Free Tier account
AWS Free Tier gives 12 months of free services including EC2 (750 hrs/month), S3 (5GB), CloudTrail, and more. Set up billing alerts immediately — accidental charges happen to beginners.
aws.amazon.com/free
aws budgets create-budget --budget-name "lab" --amount 5
2
Harden your AWS root account (Day 1)
Never use root for daily work. Create an IAM user with least-privilege policies. Enable MFA on root and all IAM users. This is the first thing every cloud security interviewer will ask about.
aws iam create-virtual-mfa-device
aws iam enable-mfa-device --user-name admin --serial-number ARN
3
Enable CloudTrail + GuardDuty
CloudTrail logs every API call in your account — the equivalent of an audit log. GuardDuty is AWS's built-in threat detection (free 30-day trial). Enable both in every region you use.
aws cloudtrail create-trail --name lab-trail --s3-bucket-name lab-logs
aws guardduty create-detector --enable
4
Practice CloudGoat (intentionally vulnerable AWS)
CloudGoat by Rhino Security Labs is the "Metasploitable for AWS." It deploys misconfigured environments for attack/defense practice: privilege escalation, S3 data exfil, lambda abuse, and more.
git clone https://github.com/RhinoSecurityLabs/cloudgoat
pip3 install -r requirements.txt
./cloudgoat.py create iam_privesc_by_attachment
5
Set up Azure free account + Microsoft Sentinel
Azure gives $200 credit + 12 months of free services. Enable Microsoft Sentinel (cloud SIEM) and Defender for Cloud. Connect your Azure activity logs. Sentinel uses KQL for queries — a critical skill for Azure security jobs.
portal.azure.com → Microsoft Sentinel → Create workspace
az sentinel onboarding-state create -g lab-rg -w lab-workspace
6
Use Prowler for cloud security auditing
Prowler is a free open-source cloud security assessment tool. Run it against your AWS account to identify misconfigurations and benchmark against CIS controls. Great for interview talking points.
pip3 install prowler
prowler aws --compliance cis_2.0_aws
Tools Used:AWS Free TierAzure FreeCloudGoatProwlerMicrosoft SentinelGuardDuty

About InfoSecDeck
AJ
AJ Poole
IAM Engineer  ·  10 Years Experience

I started my career at the help desk. Ten years later, I've spent eight of them working in Identity & Access Management — and I still get messages from friends, coworkers, and former colleagues asking the same questions: How do I break into cybersecurity? What certs should I get first? What domain is right for me?

Every time I tried to point people to a resource, I ran into the same problem: there wasn't one place that laid it all out clearly — the career paths, the honest cert guidance, the salary data, the domain breakdowns. Everything was scattered, outdated, or trying to sell something. So I built InfoSecDeck.

This site is a compilation of the advice I was given throughout my own career, combined with best practices and recommendations from top sources across the industry. My goal was simple: create the universal resource I wish I could have shared with everyone who reached out to me — a single place where someone serious about cybersecurity can find everything they need, without the noise.

Everything on InfoSecDeck is independently researched and maintained. No vendor bias, no fluff — just real, practitioner-tested guidance.
